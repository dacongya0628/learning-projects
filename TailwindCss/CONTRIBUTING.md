# 贡献指南

感谢您对 LearnTailwindCSS 项目的关注！欢迎提交 Issue 和 Pull Request。

## 🤝 如何贡献

### 报告问题
- 使用 GitHub Issues 报告 bug
- 提供详细的复现步骤
- 包含错误信息和环境信息

### 提交功能
1. Fork 项目到您的 GitHub 账户
2. 创建功能分支：`git checkout -b feature/amazing-feature`
3. 提交更改：`git commit -m 'Add some amazing feature'`
4. 推送分支：`git push origin feature/amazing-feature`
5. 创建 Pull Request

## 📝 代码规范

### JavaScript 代码风格
- 使用 ES6+ 语法
- 使用有意义的变量和函数名
- 添加适当的注释
- 保持代码简洁和可读性

### 提交信息规范
```
type(scope): description

[optional body]

[optional footer]
```

类型说明：
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

示例：
```
feat(generateCss): 添加新的 CSS 属性支持

- 添加 width 和 height 属性映射
- 更新配置文件结构
- 添加相关测试用例
```

## 🧪 测试

在提交 PR 前，请确保：
- 代码能够正常运行
- 构建过程没有错误
- 新功能有相应的测试用例

## 📚 文档

- 更新相关的文档
- 添加使用示例
- 确保文档的准确性

## 🎯 贡献方向

我们欢迎以下类型的贡献：

1. **新功能支持**
   - 添加新的 CSS 属性
   - 支持更多的设计令牌
   - 增加新的变体支持

2. **性能优化**
   - 优化文件扫描性能
   - 改进 CSS 生成算法
   - 减少内存使用

3. **平台兼容性**
   - 增加对更多框架的支持
   - 改进移动端适配
   - 优化小程序支持

4. **文档改进**
   - 完善 API 文档
   - 添加更多使用示例
   - 翻译文档

5. **测试覆盖**
   - 添加单元测试
   - 集成测试
   - 端到端测试

## 📞 联系方式

如有任何问题，请通过以下方式联系：
- GitHub Issues
- Pull Request 讨论

---

再次感谢您的贡献！🙏
