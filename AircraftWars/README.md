# AircraftWars
> 一个用原生 JavaScript 和 Canvas 实现的飞机大战学习项目。

这个仓库通过一个简化版纵向射击小游戏，练习浏览器原生图形绘制、游戏循环、输入控制、碰撞检测和资源预加载。项目没有引入构建工具或框架，适合拿来直接阅读源码、理解基础游戏逻辑是如何拆分与协作的。

## 🎯 学习目标

- 理解 `canvas` 2D 渲染上下文的基础绘制方式
- 练习用 `requestAnimationFrame` 组织浏览器端游戏主循环
- 掌握键盘输入与触摸输入的统一接入思路
- 熟悉实体更新、子弹生成、敌机刷出与碰撞检测的基本流程
- 通过拆分类模块的方式，练习原生 JavaScript 项目结构设计

- 移动端可通过触摸拖动飞机位置
- 游戏开始后，飞机会自动发射子弹
- 敌机会从屏幕上方随机刷出，命中后会播放爆炸效果

- HP 归零后弹出游戏结束面板，显示击杀数，可重新开始或返回首页

## 🧠 设计要点

### 游戏主循环

- `Timer` 对 `requestAnimationFrame` 做了轻量封装，用任务列表驱动每一帧更新
- `Main.gameStar()` 中注册主循环后，会持续执行清屏、更新实体、碰撞检测和重新绘制
- 游戏结束后通过 `isGameOver` 标志停止更新，并调用 `timer.stop()` 停止循环

### 实体拆分

- `Plane` 负责玩家位置更新、边界限制、自动发射子弹与无敌状态管理
- `Enemy` 负责敌机出生位置、下落移动和敌机开火
- `Bullet` 负责子弹位移与绘制
- `Explosion` 负责爆炸淡出和尺寸变化
- `HpBar` 作为独立血条组件，支持渐变填充、缓动动画、受击闪烁与抖动

### 输入与边界控制

- 键盘输入统一在 `Controller` 中维护方向状态，`Plane.update()` 只关心状态结果
- 触摸输入直接更新飞机坐标，适合移动端快速验证交互
- 玩家移动时做了边界裁剪，避免飞机飞出屏幕

### 画布适配

- `Main.resizeCanvas()` 会同步逻辑尺寸与物理像素尺寸
- 使用 `devicePixelRatio` 提升高分屏下的绘制清晰度
- `css/index.css` 让背景图与画布始终铺满视口

### 动态难度系统

- 敌机生成间隔和玩家射击间隔均随击杀数动态调整
- 每击杀 100 架敌机提升一个难度等级
- 敌机生成：间隔下限每级减 30ms，上限每级减 80ms（最低 50ms）
- 玩家射速：间隔每级减 20ms（最低 100ms）
- 双方同步增强，保持游戏的可玩性和挑战性

### 无敌与受击反馈

- 受击后进入 3 秒无敌状态，期间战机以 100ms 间隔闪烁
- 无敌期间不扣血、不重复触发受击
- 血条同时提供闪烁、红色覆盖层和抖动三种受击反馈

## 🛠 开发指引

### 核心模块

1. `js/Main.js`
   负责初始化画布、加载图片、启动游戏、驱动所有实体更新与碰撞检测，以及游戏结束管理。
2. `js/Plane.js` / `js/Enemy.js`
   负责玩家与敌机两类核心实体的移动、开火和绘制数据。
3. `js/Bullet.js` / `js/Explosion.js` / `js/HpBar.js`
   负责战斗表现层元素，便于后续继续扩展动画与命中特效。
4. `js/Controller.js` / `js/Timer.js`
   负责输入和游戏循环，为后续新增玩法提供通用基础设施。

### 适合继续扩展的方向

- 增加计分板、逃脱计数和连击奖励
- 补充暂停功能和设置界面
- 区分不同敌机类型、子弹类型或掉落机制
- 增加音效、连续帧动画和更完整的资源管理
- 引入对象池优化子弹和敌机的频繁创建/销毁
