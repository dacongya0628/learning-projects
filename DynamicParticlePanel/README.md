# DynamicParticlePanel
> 一个用原生 Canvas 实现全屏动态粒子连线效果的前端学习项目。

这个项目通过一页纯静态页面，练习了 `canvas` 绘制、粒子运动、距离连线、窗口自适应以及简单的性能优化思路。代码规模不大，但已经覆盖了一个动态背景面板从初始化、渲染到 resize 重算的完整链路，适合作为前端动画与可视化入门练习。

## 🎯 学习目标

- 理解原生 `CanvasRenderingContext2D` 的基础绘制流程
- 练习粒子节点的位置更新、速度反弹与逐帧动画循环
- 学习基于距离阈值的粒子连线效果实现方式
- 体验通过网格分区减少无效粒子配对计算的优化思路
- 了解浏览器窗口尺寸变化时的画布重算与设备像素比适配

## ⚙️ 设计要点

### 1. 画布尺寸与清晰度适配

项目没有直接把逻辑尺寸等同于物理像素，而是通过 `devicePixelRatio` 设置 `canvas.width` / `canvas.height`，再调用 `setTransform` 让绘制坐标继续保持在 CSS 像素体系中。这样在高分屏设备上仍能保持较清晰的粒子和连线效果。

### 2. 粒子状态模型

每个粒子节点都包含以下核心属性：

- `x` / `y`：当前位置
- `vx` / `vy`：速度向量
- `radius`：粒子半径
- `id`：用于避免重复连线判断

这种数据结构足以支撑基础动画，也方便后续扩展更多视觉属性，例如颜色、透明度或交互状态。

### 3. 网格分区减少计算量

如果直接让所有粒子两两比较，计算量会随着粒子数量快速增长。当前实现先把粒子放入网格容器，再只遍历当前格子及其邻近格子中的粒子，从而减少不必要的距离判断。这是这个学习项目里比较关键的性能优化点。

### 4. Resize 节流处理

窗口拖动过程中，项目没有在每次 `resize` 事件里立刻重算，而是借助 `requestAnimationFrame` 做一次轻量节流，避免高频触发带来的重复计算。

## 🧭 开发指北

### 核心模块

1. **初始化阶段**：`init()` 获取画布上下文、绑定事件，并启动首帧渲染与动画循环。
2. **画布管理阶段**：`resizeCanvas()` 负责同步视口尺寸与 DPR，`handleResize()` 负责 resize 后的数据重建或缩放。
3. **粒子更新阶段**：`nodeMove()` 逐帧更新粒子位置并处理碰撞反弹。
4. **渲染阶段**：`drawNodes()` 负责绘制粒子，`drawLines()` 负责绘制连接线，`renderFrame()` 统一组织单帧绘制。
