# 大葱鸭随机抽奖机 · 本地使用说明

本说明面向**拿到本程序后想在个人电脑上使用**的用户，**Windows** 与 **macOS** 均可按下列步骤操作。

---

## 最简单：双击打开（推荐，无需安装 Node）

适用于：对方只拿到 **`dist` 文件夹**（由开发者在本机执行 `npm run build` 生成）。

1. 把整个 **`dist`** 文件夹拷到对方电脑任意位置（不要只拷走一半文件；至少保留 **`index.html`**，建议 **整夹原样** 一起发，内含 `favicon.svg` 等）。
2. **双击 `index.html`**，用系统默认浏览器打开；若没有自动用浏览器打开，可 **右键 → 打开方式 → Chrome / Edge / Safari**。
3. 建议使用 **Chrome、Edge** 或 **Safari** 等较新版本；若双击后页面空白，可改用下文「方式二」通过本地服务打开。

**说明：**

- 样式里若引用了网络字体，**完全离线**时可能短暂加载失败，会自动退回系统字体，不影响使用。
- 这是单文件打包后的页面，**主程序都在 `index.html` 里**；发给别人时 **ZIP 压缩整个 `dist` 文件夹** 最省事。

---

## 重要提示（备选方式才需要）

若双击无法使用，或你希望用固定端口访问，可在电脑里启动一个**本地网页服务**，再用浏览器访问 `http://localhost:端口`（见下文方式二、三）。

---

## 准备工作：安装 Node.js（仅在使用源码或命令行服务时需要）

Node.js 在 Windows 与 Mac 上安装方式相同：

1. 打开官网：<https://nodejs.org/>
2. 下载 **LTS（长期支持版）** 安装包。
3. **Windows**：运行 `.msi`，一路下一步即可。  
   **macOS**：运行 `.pkg`，或若已安装 [Homebrew](https://brew.sh/) 可执行：`brew install node`
4. 安装完成后，**重新打开**终端，输入下面命令确认有版本号输出：
   ```bash
   node -v
   npm -v
   ```

## 方式一：使用完整项目文件夹（适合有源码包时）

适用于：压缩包内含有 `package.json`、`src` 等完整目录。

### 打开终端

| 系统 | 操作 |
|------|------|
| **Windows** | 按 `Win` 搜索 **PowerShell** 或 **终端**，打开。 |
| **macOS** | 打开 **启动台 → 其他 → 终端**，或搜索 **Terminal**。 |

### 进入项目所在文件夹

把 `LotteryGame`（或你解压后的文件夹名）放到例如桌面，然后在终端里执行（路径按你实际位置修改）：

**Windows（PowerShell）示例：**

```powershell
cd $env:USERPROFILE\Desktop\LotteryGame
```

**macOS 示例：**

```bash
cd ~/Desktop/LotteryGame
```

> 小技巧：在文件夹窗口里，Windows 可按住 `Shift` 右键空白处选择「在此处打开 PowerShell 窗口」；Mac 可在访达中右键文件夹按住 `Option`，选择「将路径拷贝为路径名称」，再粘贴到 `cd ` 后面。

### 安装依赖并运行

```bash
npm install
npm run build
npm run preview
```

终端里会出现类似地址：**http://localhost:4173**  
用 **Chrome、Edge、Safari** 等浏览器打开该地址即可使用。

若希望改代码时自动刷新，可用开发模式（可选）：

```bash
npm install
npm run dev
```

按终端提示打开 **http://localhost:5173**。

## 方式二：只有 `dist` 文件夹，用命令行打开（备选）

适用于：双击 `index.html` 在你电脑上不可用，或你更习惯用浏览器地址栏访问。

### 选项 A：用 Node 临时起一个网站（与系统无关）

先安装好 Node.js，在终端中进入 **`dist` 的上一级目录**（即能看到 `dist` 文件夹的那一层），执行：

```bash
npx --yes serve dist -l 4173
```

浏览器访问：**http://localhost:4173**

若你当前就在 `dist` 文件夹内部，可执行：

```bash
npx --yes serve . -l 4173
```

### 选项 B：用 Python 起静态服务

**macOS** 一般自带 Python 3，在终端进入 `dist` 目录后：

```bash
cd /你的路径/dist
python3 -m http.server 8080
```

浏览器访问：**http://localhost:8080**

**Windows** 若已安装 Python，在 **命令提示符** 或 **PowerShell** 中进入 `dist` 后尝试：

```powershell
cd C:\你的路径\dist
python -m http.server 8080
```

若提示找不到命令，可到 <https://www.python.org/> 安装 Python，安装时勾选 **Add Python to PATH**，然后重开终端再试。

## 关闭程序

在运行 `npm run preview`、`npx serve` 或 `python -m http.server` 的终端窗口里按 **`Ctrl + C`**（Mac 同样为 **Ctrl + C**）即可停止服务，关闭浏览器标签页不影响已停止的服务。

## 常见问题

| 现象 | 处理建议 |
|------|----------|
| 提示 `npm` 不是内部或外部命令 | 未安装 Node 或未重启终端，请安装 LTS 并重开终端。 |
| 端口被占用 | 把 `4173`、`8080` 改成其他数字（如 `4174`、`8081`），地址里端口一起改。 |
| Mac 提示无法打开来自未识别开发者的安装包 | 系统设置 → 隐私与安全性 → 仍要打开。 |
| 想给同一 WiFi 下的手机访问 | 需电脑防火墙允许该端口；浏览器地址把 `localhost` 换成电脑的局域网 IP（进阶，可选）。 |
