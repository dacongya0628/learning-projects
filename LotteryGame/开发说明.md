## 2. 技术选型：程序是怎么「做出来」的

### 2.1 为什么选择这些技术

- **Vite**：开发时冷启动快、热更新方便；生产构建基于 Rollup，输出静态资源。
- **TypeScript**：在浏览器端逻辑（校验、抽奖顺序、异步控制）上提供类型约束，减少低级错误。
- **无框架（Vanilla DOM）**：单页体量小，无 React/Vue 运行时；界面通过 `innerHTML` 模板字符串挂载，事件在 `main.ts` 绑定。
- **纯 CSS 动画**：揭晓动效用 `requestAnimationFrame` 切换数字 + CSS `@keyframes`（滚动感、弹跳、粒子层），不引入动画库。

### 2.2 从 0 到可运行的步骤（概念上）

1. 使用 `npm create vite@latest` 选择 **Vanilla + TypeScript** 模板初始化工程。
2. 删除模板演示代码，保留 `index.html` 中 `#app` 挂载点与 `src/main.ts` 入口。
3. 拆分模块：`state`（类型与默认数据）、`draw`（数学与校验）、`ui`（DOM 与动画）、`main`（流程与事件）。
4. 编写 `styles.css` 做响应式与主题。
5. 增加 `vite-plugin-singlefile` 与 `base: './'`，使 `npm run build` 产出可双击打开的 **单文件 HTML**（见第 10 节）。

## 5. 整体架构与数据流

### 5.1 模块依赖关系

```
index.html
    └── main.ts
            ├── styles.css
            ├── draw.ts  ←── state.ts (类型)
            └── ui.ts    ←── state.ts, version.ts
```

- **`main.ts`** 是唯一聚合点：调用 `mountShell` 生成页面，注册所有按钮逻辑，执行 `runLottery`。
- **`draw.ts`** 不操作 DOM，只负责「规则是否合法」和「号码队列怎么来」。
- **`ui.ts`** 负责「长什么样」和「怎么动」；`main.ts` 通过 `Elements` 引用具体节点。

### 5.2 用户流程（与代码对应）

1. **设置阶段**（`phase-setup`）：用户填区间、等级列表 → 点「进入抽奖舞台」→ `validateDraw` 通过 → `setPhase(..., 'stage')`。
2. **舞台阶段**（`phase-stage`）：点「开始抽奖」→ `runLottery` 按等级循环，每次 `playRevealAnimation` 再写入 `results` 并 `renderResults`。
3. **返回设置**：`goToSetup` 中止抽奖、清空结果、回到 `phase-setup`。

### 5.3 简化数据流（文字）

- 配置来源：DOM 输入 → `readMinMax` + `readTiers` → `PrizeTier[]`。
- 抽奖前：`prepareDrawQueue(min,max)` → 得到 **洗牌后的数组**，之后每次 `pop()` 取一个号（不放回）。
- 结果：`ResultEntry[]` 存在 `main.ts` 模块级变量 `results` 中，每次揭晓 push 一条并刷新右侧列表。

## 6. 数据模型（`src/state.ts`）

| 名称 | 含义 |
|------|------|
| `PrizeTier` | 一个中奖等级：`id`（稳定键，用于结果分组）、`name`（展示名）、`count`（该等级要抽几个） |
| `ResultEntry` | 一条中奖记录：`tierId`、`tierName`、抽中的 `number` |
| `DrawPhase` | 类型已定义（`idle` / `drawing` / `completed`），当前实现主要靠 UI 与 `drawController` 表达状态，未单独维护状态机变量 |
| `createId()` | 用 `Math.random().toString(36)` 生成短字符串，给每个等级行一个稳定 `data-tier-id` |
| `defaultTiers()` | 初始三条：一等奖 1 人、二等奖 3 人、三等奖 5 人 |

## 7. 抽奖与校验逻辑（`src/draw.ts`）

### 7.1 `validateDraw(min, max, tiers)`

按顺序检查：

- `min`/`max` 为有限、整数；`min <= max`；区间长度 `max - min + 1` 有效。
- 至少一个等级；所有等级名称非空；每个 `count` 至少为 1。
- **总人数** `sumTierCounts(tiers)` 不能超过区间内号码个数（不放回的必要条件）。

通过返回 `null`，否则返回**中文错误字符串**（由 `setError` 显示）。

### 7.2 `buildRange` 与 `shuffle`

- `buildRange(min, max)`：生成 `[min, min+1, …, max]`。
- `shuffle`：**Fisher–Yates** 洗牌，返回新数组，不修改入参。

### 7.3 `prepareDrawQueue`

```ts
shuffle(buildRange(min, max))
```

即：**整段区间先打乱**，后面从数组尾部 `pop()` 取号。等价于「随机不放回」：每个号码在整次抽奖中最多出现一次。

### 7.4 为何用 `pop()` 而不是 `shift()`

数组已洗牌，从尾部弹出与从头部弹出在随机性上等价；`pop()` 实现简单、O(1)。

## 8. 主流程与并发控制（`src/main.ts`）

### 8.1 常量

- `ROLL_MS`（约 1100ms）：数字快速滚动伪装「正在抽」的时长。
- `PAUSE_BETWEEN_MS`（约 550ms）：两次揭晓之间的停顿。

### 8.2 `runLottery(els, signal)`

1. 再次读取区间与等级（与舞台页是否可见无关，数据仍在 DOM 中）。
2. `validateDraw`；失败则 `setError` 并 **return**（不清空已有结果，除非前面已清空）。
3. 清空 `results`，重绘空列表。
4. `queue = prepareDrawQueue(min, max)`。
5. 双重循环：**外层**按 `tiers` 顺序，**内层**每个 `tier.count` 次：
   - `queue.pop()` 得到下一个中奖号码；若 `undefined` 则报错返回（理论上校验通过不应发生）。
   - 更新舞台标题、进度文案。
   - `await playRevealAnimation(...)`。
   - `results.push` + `renderResults`。
   - 若非最后一次，则 `await delay(..., signal)`。
6. 全部结束后更新「全部完成」类文案。

任意一步若 `signal.aborted`（用户点了「重置本轮」或「返回抽奖设置」触发 `abort()`），提前退出。

### 8.3 `AbortController`

- 点击「开始抽奖」时 `new AbortController()`，把 `signal` 传给 `runLottery`、`delay`、`playRevealAnimation`。
- 「重置本轮」「返回抽奖设置」会 `drawController?.abort()`，使正在 `await` 的 Promise 以 `AbortError` 结束；`main.ts` 在 `catch` 里区分是否中止并提示。

### 8.4 `setDrawingUiLocked`

定义在 `ui.ts`，在抽奖进行中禁用「开始」「返回设置」及所有设置区控件，**不禁用**「重置本轮」，以便随时中止。

### 8.5 `scrollIntoViewIfNarrow`

仅在视口宽度 **≤767px** 时对舞台/设置容器执行 `scrollIntoView`，避免 PC 宽屏下不必要的页面跳动。

## 9. 界面层（`src/ui.ts`）

### 9.1 `mountShell(root)`

- 将 `#app` 的 `innerHTML` 替换为**整页结构**：页头、主区域（步骤条、错误区、设置卡片、`draw-view` 内含舞台+结果）、页脚。
- 使用 `<template id="tier-template">` 存放一行等级的 HTML 片段，供 `addTierRow` 克隆。
- 页脚版本号通过模板字符串插入 `${APP_VERSION}`。
- 返回 `Elements` 对象，内含所有需要由 `main` 操作的 DOM 引用。

### 9.2 `setPhase(els, 'setup' | 'stage')`

- 给 `#app-main` 切换 class：`phase-setup` / `phase-stage`。
- CSS 中通过 **`.phase-setup .draw-view { display: none }`**、**`.phase-stage .config-card { display: none }`** 实现两步界面（具体以 `styles.css` 为准）。

### 9.3 等级列表

- `readTiers`：遍历 `.tier-row`，读名称与人数；无效行会被过滤。
- `addTierRow`：克隆模板，写入 `data-tier-id`，追加到 `ul`。
- `fillDefaultTiers`：清空后按 `defaultTiers()` 填充。
- `suggestNextTierName`：根据**当前行数+1** 生成「一等奖…十等奖」或「第 N 等级」。
- `bindTierListActions`：**事件委托**，处理上移、下移、删除（至少保留一行）。

### 9.4 `playRevealAnimation`

- 给数字节点加 `is-rolling`，用 `requestAnimationFrame` 在 `rollMs` 时间内不断把文字设为 `[min,max]` 内随机整数（**视觉**用，与真实奖池无关）。
- 时间到后显示**真实号码**，加 `is-landed`、`land-pop` 触发 CSS 弹跳。
- `stage-particles` 通过 `burst` class 做一次短粒子动画（依赖 `offsetWidth` 触发重排以重播动画）。

### 9.5 `renderResults`

- 将 `ResultEntry[]` 按 `tierId` 分组，生成每个等级下的「芯片」列表。
- 使用 `escapeHtml` 防止等级名中特殊字符破坏 HTML。

### 9.6 `delay(ms, signal?)`

可取消的 `Promise` 版 `setTimeout`，与 `AbortSignal` 联动。

## 10. 样式（`src/styles.css`）

- **设计变量**：`:root` 中粉紫配色、圆角、阴影、字体（含 Google Fonts `@import`，离线时回退系统字体）。
- **响应式**：移动优先；`draw-view` 在约 **768px** 以上双列（舞台 | 结果），以下单列。
- **阶段**：`.phase-setup` / `.phase-stage` 控制设置与舞台显隐。
- **等级行**：PC 端两列等宽输入 + 右侧按钮；移动端单行紧凑布局（详见文件内注释与媒体查询）。
- **舞台**：`.stage-display`、`.stage-number` 的滚动/落定 class 与 `ui.ts` 同步。
- **页脚**：技术支持条、定制说明、版本号的字号与对比度分层。

修改视觉时，**优先改 class 与 CSS**，一般不必改 TS 逻辑。

## 11. 单文件构建与 favicon（`vite.config.ts` / `index.html`）

- **`base: './'`**：资源使用相对路径，便于 `file://` 打开。
- **`viteSingleFile()`**：构建结束后把 JS、CSS **内联进 `index.html`**，生成以单文件为主的 `dist`（便于分发）。
- **标签图标**：`index.html` 里使用 **`data:image/svg+xml,...` 内联 SVG** 作为 `rel="icon"`，避免浏览器长期缓存旧 `favicon.svg`、或误用开发工具默认闪电图标。`public/favicon.svg` 仍保留作源文件；若只改 SVG，需同步更新 `index.html` 中的 data URI，或在项目根执行 `node scripts/encode-favicon.mjs` 生成编码串后替换 `href`。

## 12. 扩展与修改建议

| 需求 | 建议切入点 |
|------|------------|
| 改默认区间/等级 | `ui.ts` 模板里 `value`、`state.ts` 的 `defaultTiers`、`main.ts` 里「恢复默认」 |
| 改动画时长 | `main.ts` 的 `ROLL_MS`、`PAUSE_BETWEEN_MS`；`ui.ts` 里 `playRevealAnimation` 末尾 `delay(650)` |
| 持久化配置/结果 | 在 `main.ts` 读写 `localStorage`（注意隐私与容量） |
| 国际化 | 将 `ui.ts` 模板字符串与 `draw.ts` 错误文案抽成字典 |
| 部署到服务器 | `npm run build` 后上传 `dist`；若部署在子路径，需调整 `base` |
| 恢复多文件分包（利于 CDN 缓存） | 从 `vite.config.ts` 移除 `viteSingleFile`，并视情况改回 `base` |

## 13. 小结

本程序本质是：**纯前端静态页** + **一次洗牌的不放回抽样** + **async/await 串联动画** + **AbortController 取消**。无后端、无数据库；随机性来自 `Math.random` 驱动的 Fisher–Yates。掌握 `main.ts` 的 `runLottery` 与 `draw.ts` 的校验/队列，即可把握核心业务；`ui.ts` 负责呈现与交互细节。

若你修改了行为或新增模块，建议同步更新 **README.md** 与本 **开发说明.md**，便于后续自己与协作者查阅。
